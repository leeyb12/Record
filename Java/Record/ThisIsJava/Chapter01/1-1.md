# 1.1 프로그래밍 언어와 자바

## 자바 프로그래밍 입문 및 개발 환경 구축

"자바는 다양한 플랫폼에서 동작하는 강력한 프로그래밍 언어로, 개발자가 메모리 관리에 신경 쓰지 않고 핵심 기능에 집중할 수 있도록 설계되었다."

<br>

## 1장: 자바 시작하기

| 주요 내용 | 설명 |
|-----------|------|
| 자바 언어 특징 | 플랫폼 독립성, 객체 지향 프로그래밍, 메모리 자동 관리 등 | 
| 개발 환경 구축 | 운영체제별 JDK(Java Development Kit, 자바 개발 키트) 설치 방법 (윈도우, 맥) | 
| 환경 변수 설정 | 자바 실행 편의를 위한 환경 변수 설정 방법 |
| 소스 작성부터 실행까지 | 텍스트 파일로 작성 후 컴파일 및 실행, 이후 Eclipse IDE 사용법 학습 | 
| 코드 용어 및 문장 구성 | 코드 용어 이해, 주석 달기, 실행문의 종류 및 세미콜론(;) 역할 학습 |

<br>

## 컴퓨터 언어의 이해

|  구분  |  설명  |
|--------|--------|
| 기계어 (Machine Language) | 0과 1로 된 이진 코드, 컴퓨터가 직접 이해하는 언어 |
| 프로그램 언어 (Programming Language) | 사람과 컴퓨터 사이의 다리 역할, 고급 언어 및 저급 언어(어셈블리어)로 구분 |
| 고급 언어 (High-Level Language) | 사람이 이해하기 쉽도록 설계된 언어 (예: 자바, C, C++, 파이썬) |
| 저급 언어 (Low-Level Language) | 기계어에 가까운 언어, 사람이 이해하기 어려움 (예: 어셈블리어) | 

"고급 언어로 작성된 소스 파일을 컴파일러가 기계어로 변환하는 과정을 컴파일링(compiling)이라고 한다."

<br>

## 컴파일과 실행 방식 비교

| 언어 종류 | 소스 -> 변환 과정 | 실행 방식 |
|-----------|------------------|-----------|
| C, C++ | 소스 파일 -> 컴파일러 -> 기계어 파일 | 기계어 파일 직접 실행 |
| 자바 | 소스 파일 -> 컴파일러 -> 바이트 코드 파일 | 자바 가상 머신(Java Virtual Machine, JVM)이 바이트 코드를 해석하여 실행 |
| 파이썬 | 소스 파일 직접 | 인터프리터가 소스 파일을 한 줄씩 해석하여 실행 | 

- 바이트 코드(Byte Code) : 자바 컴파일러가 생성하는 중간 코드. 플랫폼 독립적이며 JVM에서 해석되어 실행된다.
- 자바 가상 머신(JVM) : 바이트 코드를 각 운영체제에 맞는 기계어로 변환하여 실행하는 가상 환경.

<br>

## 자바 언어의 역사 및 활용 분야

- 출시 연도: 1995년, 선 마이크로시스템즈에서 발표(현재 오라클(Oracle) 소유)
- 주요 활용 분야:
  - 안드로이드 앱 개발 (코틀린과 함께 사용, 자바가 여전히 널리 사용)
  - 웹 애플리케이션 개발 (현재 주된 언어 중 하나)
  - 데스크톱 애플리케이션 (속도 문제로 주로 C++ 사용, 자바는 스윙(Swing)이나 자바FX(JavaFX)를 이용)

"자바는 작성된 코드를 한 번 컴파일하면 어떤 운영체제에서도 실행할 수 있는 '플랫폼 독립성'을 가진다."

<br>

## 자바의 주요 특징

| 특징 | 설명 |
|------|------|
| 운영체제 독립성 | 한번 컴파일된 바이트 코드는 윈도우, 맥, 리눅스 등 어떤 OS에서도 실행 가능 |
| 완전한 객체 지향 프로그래밍 | 모든 것이 객체(클래스와 인스턴스)로 구성, 다른 언어에 비해 객체 지향적 특성 강함 |
| 자동 메모리 관리 | 가비지 컬렉션(Garbage Collection)을 통해 메모리 해제 자동 수행, 개발자가 메모리 관리 코드 작성 불필요 |
| 풍부한 무료 라이브러리 | 다양한 오픈 소스 라이브러리와 API가 제공되어 개발 기간 단축 가능 |

<br>

## 프로그래밍 언어 및 컴파일 구조 정리

| 용어 | 의미 및 역할 |
|------|--------------|
| 소스 파일(Source File) | 사람이 이해할 수 있는 프로그래밍 언어로 작성된 파일 |
| 컴파일러(Compiler) | 소스 파일을 기계어 또는 중간 코드(바이트 코드)로 변환하는 소프트웨어 |
| 기계어(Machine Code) | 컴퓨터가 직접 실행 가능한 이진 코드 |
| 인터프리터(Interpreter) | 소스 코드를 한 줄씩 해석하여 바로 실행하는 프로그램 |

<br>

## 자바 개발 도구 및 지원

- JDK(Java Development Kit) : 자바 개발에 필요한 컴파일러, 라이브러리, JVM 등이 포함된 도구 모음
- 오라클(Oracle) : 자바의 공식 라이선스 소유 및 JDK 배포, 기술 지원 제공
- Eclipse IDE : 자바 소스 코드 작성, 컴파일, 실행을 쉽게 할 수 있는 통합 개발 환경

<br>

## 프로그래밍 언어와 컴파일 과정의 메타포

"사람과 컴퓨터는 서로 다른 언어를 사용한다. 사람이 이해하기 쉬운 고급 언어는 컴퓨터가 직접 이해하지 못하므로, 중간 다리 역할을 하는 컴파일러가 필요하다. 이는 사람이 영어로 쓴 편지를 외국어로 통역해 주는 통역사와 같다."

<br>

## 코드 작성 시 중요한 사항

| 항목 | 설명 |
|------|------|
| 주석(Comment) | 코드 내에서 설명을 달아 다른 사람이 이해하기 쉽게 도와주는 문장 |
| 세미콜론(;) | 실행문(statement)의 끝을 표시하는 구분자, 문장 구분에 필수 |
| 실행문 종류 | 변수 선언, 할당, 조건문, 반복문 등 프로그램의 동작을 구성하는 문장 |

<br>

## 주요 용어 정리

| 용어 | 정의 |
|------|------|
| 바이트 코드(Byte Code) | 자바 컴파일러가 생성하는 중간 코드, JVM에서 실행됨 |
| 가상 머신(Virtual Machine) | 실제 하드웨어와 독립적으로 동작하는 소프트웨어 기반 컴퓨터 |
| 객체 지향 프로그래밍(Object-Oriented Programming) | 데이터를 객체로 구성하여 설계하는 프로그래밍 패러다임 |
| 가비지 컬렉션(Garbage Collection) | 더 이상 사용되지 않는 메모리를 자동으로 회수하는 자바의 메모리 관리 기법 |

<br>

## 핵심 요약

- 자바는 **플랫폼 독립적**이며 **객체 지향적**인 고급 프로그래밍 언어이다.
- 자바 소스 코드는 **바이트 코드**로 컴파일되고, 이 바이트 코드는 **JVM**에 의해 실행된다.
- 자바는 **자동 메모리 관리** 기능을 갖추어 개발자의 부담을 줄여준다.
- 다양한 운영체제에서 동일한 실행 환경을 제공하며, 풍부한 무료 라이브러리로 개발 기간을 단축할 수 있다. 
- JDK 설치 및 환경 변수 설정, Eclipse IDE 사용법 등 개발 환경 구축이 필수적이다.
